* [2026-05-13] Dev Log: Codex browser MCP wiring              :BROWSER:MCP:CODEX:

** Context
Deva already supports Claude host Chrome via =deva.sh claude -- --chrome=; Codex CLI does not consume that bridge.

** Why
Codex browser automation needs Codex MCP configuration, not Claude's host Chrome socket and profile-extension mounts.

** What
- Added =deva.sh codex --browser-mcp=.
- The flag switches default/base profile selection to =rust= because browser runtime dependencies live in the rust image.
- The flag injects a session-only Codex CLI override for =mcp_servers.playwright=.
- The flag does not edit or mount over =~/.codex/config.toml=.
- =deva.sh claude --browser-mcp= now fails with a direct error; Claude host Chrome remains =deva.sh claude -- --chrome=.
- Added =CODEX_BROWSER_MCP=true= and repeatable =CODEX_CONFIG=key=value= support for =.deva= files.
- Added dry-run shape coverage in =scripts/test-codex-browser-shape.sh=.
- Documented the distinction in =docs/how-it-works.md= and =docs/troubleshooting.md=.

** How
*** Commands
#+BEGIN_SRC bash
bash scripts/test-codex-browser-shape.sh
bash scripts/test-mount-shape.sh
bash -n deva.sh agents/codex.sh scripts/test-codex-browser-shape.sh scripts/test-mount-shape.sh

#+END_SRC
